How this was built

The system prompt

Aria's "brain" is a system prompt sent with every API call. It defines her as a warm, precise generalist assistant, asks her to match the user's format needs (quick answer vs. structured breakdown), to say plainly when she's unsure rather than guessing, and to handle three edge cases explicitly: irrelevant or empty input (ask a clarifying question instead of guessing), missing context (ask one focused follow-up), and abusive or harmful requests (decline briefly and redirect). Keeping scope broad but the *behavior* rules tight is what makes a general-purpose assistant feel reliable instead of generic.

Why this UI

A universal assistant has no single "shape" of output, so the interface leans on a flexible chat layout instead of a fixed dashboard — but with small, purposeful touches: quick-start chips for the blank-page moment, a file-attach affordance since inputs can be text or documents, and light markdown rendering so structured answers (lists, headers, code) still look considered rather than dumping raw text.

Extending it

Ideas for growing this: add persistent memory by writing conversation summaries to window.storage between sessions; add tools (web search, calculator) by defining them in the API request's tools array and handling tool_use blocks in the response; or turn this into a multi-step flow by having Aria ask a fixed sequence of clarifying questions before her first real answer, similar to an intake form.
