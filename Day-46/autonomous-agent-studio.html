5. Architecture, extensions & further prompts

    

      Architecture: Planner scopes the refactor goals once. Executor produces the first draft. Then a real loop
      calls Evaluator → Critic → Improver each round, with a fresh live API call every time —
      no cached or canned scoring. Each round's evaluation + critique is threaded into the next Improver call, and the
      Evaluator always re-scores the current draft against the rubric. After every round the app checks, in order:
      plateau (score gain < 1.5 for two rounds), threshold (score ≥ your target), then the hard cap as a safety-only
      fallback. Whichever fires first ends the loop and hands the final draft to the Final Reviewer.
    

    

      Extension ideas: add a Safety Monitor agent that vetoes unsafe code changes; add multi-file context via a
      Memory Manager that persists snippets across sessions; add a Planner→Executor loop of its own for multi-step
      refactors; swap the rubric for language-specific linters combined with the LLM judge; add human-in-the-loop
      approval before the Final Reviewer signs off.
    

    

      Further prompts to try: "Turn this into a two-agent debate system where Critic and Improver argue before
      converging," "Add a Safety Monitor that can halt the loop," "Persist iteration history to localStorage so runs
      survive a refresh," "Generalize this studio to any workflow, not just code review, by making the rubric and
      stopping condition configurable at runtime."
